# ConsoleApplication

ConsoleApplication — консольное приложение для учета запчастей на складе сервисного центра и регистрации заказов на ремонт. Проект разработан в рамках учебного задания с использованием JDBC API без применения ORM (Hibernate/JPA) и реализует паттерн DAO (Data Access Object).

---

## Описание проекта

Приложение предназначено для:

* ведения учета запчастей на складе;
* поиска запчастей по названию;
* оформления заказов на ремонт;
* транзакционного списания запчастей;
* фильтрации заказов по статусу;
* логирования ошибок при работе с базой данных.

Проект демонстрирует работу с базой данных через JDBC, ручное управление транзакциями и разделение архитектуры на слои.

---

## Архитектура

В проекте реализован паттерн DAO, обеспечивающий разделение бизнес-логики и слоя доступа к данным.

### Структура пакетов

* model — сущности Part и Order
* dao — интерфейсы доступа к данным
* dao.impl — реализация DAO через JDBC
* service — бизнес-логика и управление транзакциями
* util — подключение к базе данных
* view — консольное меню

Такое разделение повышает читаемость, масштабируемость и поддерживаемость проекта.

---

## Реализация транзакций

Процесс оформления заказа выполняется в одной транзакции:

1. Уменьшается количество запчастей на складе
2. Создается запись в таблице orders
3. При успешном выполнении вызывается commit()
4. При ошибке выполняется rollback()

Для управления транзакцией используется:

```
connection.setAutoCommit(false);
```

## Сборка и запуск

1. Выполнить schema.sql для создания таблиц

2. Собрать проект:

   ```
   mvn clean install
   ```

3. Запустить основной класс приложения
